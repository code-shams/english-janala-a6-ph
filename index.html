Frequently Asked Questions

What is the difference between var, let, and const?
  • var : var is the old way of declaring a variable in JavaScript. Var is function scoped. It can be redeclared. It is not recommended anymore.
  • let : let is the new way of declaring a variable is JavaScript. Let is block scoped. Unlike var, it cannot be redeclared within the same scope. But it can be updated.
  • const : const is just like let. The only difference is that it is immutable. It's value cannot be reassigned.
What is the difference between map(), forEach(), and filter()?
  • map(): This array method is used to create a new modified version of the array. The map() method always returns a new array with the modified elements, while the original array remains unchanged.
  • forEach(): This array method Just loops through the array and performs an action on each element. It does not return anything.
  • filter(): This array method returns a new filtered array. It checks if an element from the original array passes a certain condition. It creates a new array with the elements from the original array that passes the condition while the original array remains intact.
Explain arrow functions and how they are different from regular functions?
  • Arrow Functions: A shorter way to write functions in JavaScript using the "=>" syntax. They are great for short, simple functions. They don't get hoisted if stored as a variable.
  • Regular Functions: Obviously, the syntax is vastly different. "function" keyword is used to declare a regular function. Regular function declarations get hoisted. It's less concise and more suitable for object-oriented use cases.
How do JavaScript Promises work?

A Promise is an object representing the eventual completion (or failure) of an asynchronous operation.
States of a Promise :
Pending: The operation is still ongoing.
Fulfilled: The operation completed successfully.
Rejected: The operation failed.
To handle a promise we can use ".then" chaining method or we can use async/await functional approach.

How does closures work in JavaScript?

A closure is created when a function is defined inside another function, allowing the inner function to access variables from the outer function, even after the outer function has finished executing. If the variables declared on the outer function are also used inside the inner function then a reference of those variable are stored inside the inner function. This allows the inner function to use or modify those variables whenever it is called.
